Form fields

This section is JAWS

When you land on an input with a screen‐reader it will announce the content of the associated label. This is also reflected in the form controls listing dialog for JAWS.

There are three form fields on this page (with the exception of the button).

The first one (first name) is marked up correctly. You will hear the label read out as you move across it but also again when you reach the input itself.

The second one (last name) is not marked up correctly, the label is not associated with the input. But you will hear the same information as the first field. JAWS does something that other screen‐readers do not. If a field does not have a label associated with it, then it will assume the text (if any) above it is the label. Sometimes this works as in this example.

However, if you go the the third example (Reference number) you will see what can happen when this doesn't go quite right. Here neither the label nor the hint are associated with the input. JAWS takes the copy immedialy above the input as the label, which means the user isn't clear on what is being asked.

Just as we can navigate by headings, we can also navigate by form fields (including buttons). Use f to move from field to field, adding shift to reverse the direction.

This guessing behaviour is only seen in JAWS. Other screen‐readers would see all but the first field as unlabelled.

An easy way to see if a label is associated with a field is to click on the label. If the field gets focus then the association is present.

For example A123456

Because JAWS is what is called a “modal screen‐reader” you will need to enter “forms mode” to type in the form field. Forms mode tells JAWS you want to type now rather than use shortcut keys, so hitting h will type the letter, not go to the next heading.

To enter forms mode, navigate to a form input and use Modifier key + space. You will hear an audible tone and then you can type as normal in the text input.

To exit forms mode, repeat the use of Modifier key + space.

When testing, check each input has a label associated with it.
