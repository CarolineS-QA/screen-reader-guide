Links

This section is NVDA

As we've already seen, screen‐readers can extract all occurrences of certain elements like headings and links into a list for the user to choose from. If you have two links with different urls but with the same link text, the user might not be able to determine which one they should click on.

Below are two lists.

Open the elements dialog Modifier key + F7 to view all the links on this page. Note that the list of links will scroll to show you where you are currently on the page, so depending on where you are in the page you may need to scroll the list to see the first links.

The first list has standard links. When you view these in the dialog you won't know which Change or Remove link refers to which person. They all seem identical.

Poorly marked-up links

  • Jo Monroe Change Remove
  • Bob Hoskins Change Remove

In the second list we've added some hidden contextual information to make the links more explanatory. Now the user can clearly see what each link does. Now they can action them knowing what the link will do.

Well marked-up links

  • Sydney Russell Change Change Sydney Russell Remove Remove Sydney Russell from the list
  • Michael Cain Change Change Michael Cain Remove Remove Michael Cain from the list

When you are on a page you can use Tab to jump to the next focussable item (including links), u to jump to the next unvisited link, or v to move to the next visited link, adding in shift to reverse the direction.

Check all the links on your page in the links listing. Can you tell what each one will do without looking at the page? Add some contextual copy if you cannot.
