Form fields

This section is Voiceover for Mac

When you land on an input with a screen-reader it will announce the content of the associated label. This is also reflected in the Rotor for Voiceover.

Navigate to the two fields on this page. The text above each one read out similarly but notice how the first input says “First name, edit text” whilst the second one only says “Edit text”. You can also see this difference in the Rotor under the Form controls category - open the Rotor with Control + Alt + u.

The reason for this is that the second label is lacking a for attribute which ties the label to the input. Effectively, as far as the screen-reader is concerned, the input does not have a label.

To enter text in a field, just type as you normally would. Voiceover will read out your entry as you type.

Just as we can navigate by headings, we can also navigate by form fields (including buttons). Use Control + Alt + Command + j to move from field to field, adding shift to reverse the direction.

When testing, check each input has a label associated with it.
