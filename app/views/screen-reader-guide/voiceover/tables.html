Tables

This section is Voiceover for Mac

You should only use tables to present related data and help users compare information.

Tables can be quite complex elements to navigate and often hold a lot of data the user will need to process. We can make that much easier by ensuring tables are marked up correctly.

  • use th for headings to associate the data with the correct heading
  • add scope attributes to headings where appropriate
  • use thead and tbody
  • add a caption to indicate the contents

There are two identically presented tables on this page.

Open the Rotor Control + Alt + u and use the left/right arrow keys to move to the Tables category.

You'll see the first table is unnamed, whilst the second is labelled using the caption element. This is especially important where there might be more than one table on the page as this will assist users in finding the data they want.

Tables without captions mean the user has to navigate the complex data and try to work out what it means.

To navigate a table you use the normal Control + Alt and left/right arrow keys to progress through the table in source-code order, but you can also use Control + Alt and up/down arrow keys to move vertically.

On the first table you will hear the number of the column and row you are on read out along with the contents of the cell you are in, but there is no distinction between any of the cells.

Poorly marked-up table

Date Amount Status
June 2020 £109.80 Complete
July 2020 £155 Complete
August 2020 £145.40 Pending

Now try the second table.

As you move around you get some additional information.

As you move from column to column you will hear the column heading announced and as you move from row to row you will hear the row heading announced.

Correctly marking up your headings creates relationships between the data and will ensure the user is able to locate themselves in the table and be sure they are getting the correct information.

Well marked-up table

Dates and amounts
Date Amount Status
June 2020 £109.80 Complete
July 2020 £155 Complete
August 2020 £145.40 Pending

When you are on a page you can use Control + Alt + Command + t to jump from one table to the next table, adding in shift to reverse the direction

Give tables captions to help identify what the table is about. Correctly mark up your tables to give meaningful associations between your data.
